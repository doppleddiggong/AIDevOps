# Self-Assessment Report 생성

**참고**: `AgentRule/Self-Assessment_Agent_SOP.md`

## 실행 내용

1. Git 커밋 히스토리 전체 분석
2. DevLog 데이터 참조 (존재 시)
3. 정량 지표 기반 객관적 평가
4. 업계 표준 대비 냉정한 분석
5. 구체적 개선 과제 제시

---

## 🚨 중요: 평가 원칙

### ❌ 금지 사항
- **과장 금지**: "우수", "탁월", "뛰어남" 등 최소화
- **칭찬 일색 금지**: 약점과 개선점을 반드시 명시
- **추측 금지**: 로그에 없는 내용은 평가하지 않음
- **별점 인플레이션 금지**: 3점이 평균, 4점은 우수, 5점은 예외적

### ✅ 필수 사항
- **냉정한 평가**: 감정 배제, 사실 중심
- **상대 평가**: 업계 표준, 시니어 수준과 비교
- **구체적 수치**: 정량 지표로 증빙
- **약점 명시**: 개선 과제 최소 3개 이상
- **실패 강조**: 성공보다 실패 사례에서 학습

## 작업 지시

### 1. 분석 기간 설정

**기본값**: 최근 3개월 (현재일 기준 -90일)

**이유**: 3개월이 개인 역량 평가에 적절한 기간
- 너무 짧으면: 일시적 활동으로 왜곡
- 너무 길면: 최신 역량 반영 안됨

**사용자 지정 가능**:
```bash
# 특정 기간
--since YYYY-MM-DD --until YYYY-MM-DD

# 전체 프로젝트
--full
```

### 2. 데이터 수집

#### 2.1 Git 커밋 데이터

```bash
# 기본 커밋 로그
git log --all --no-merges \
  --since="90 days ago" \
  --pretty=format:"%H|%an|%ae|%ad|%s" \
  --date=iso

# 변경량 통계
git log --all --no-merges \
  --since="90 days ago" \
  --numstat \
  --pretty=format:"%H"

# 활동 일수
git log --all --no-merges \
  --since="90 days ago" \
  --pretty=format:"%ad" \
  --date=short | sort -u | wc -l

# 커밋 타입 분포
git log --all --no-merges \
  --since="90 days ago" \
  --pretty=format:"%s" | \
  grep -oE "^[a-z]+(\(.+\))?: " | \
  cut -d: -f1 | sort | uniq -c

# 평균 커밋 크기
git log --all --no-merges \
  --since="90 days ago" \
  --shortstat | \
  grep -oE "[0-9]+ insertion|[0-9]+ deletion"
```

#### 2.2 DevLog 참조 (선택)

```bash
# DevLog 파일 확인
ls Documents/DevLog/Daily/*.md 2>/dev/null | wc -l
ls Documents/DevLog/Weekly/*.md 2>/dev/null | wc -l
```

DevLog가 있으면:
- 주요 성과 추출
- 문제 해결 사례 참조
- 기술적 학습 내용 확인

## ⚠️ 평가 기준 안내

본 리포트는 **객관적 데이터 기반 냉정한 평가**를 원칙으로 합니다.

**별점 기준** (5점 만점):
- **1점**: 심각한 문제, 즉시 개선 필요
- **2점**: 기대 이하, 상당한 개선 필요
- **3점**: 평균 수준, 일반적 기대치 충족
- **4점**: 평균 이상, 명확한 강점 보유
- **5점**: 탁월한 수준, 업계 상위 10%

**중요**: 3점이 정상이며, 4-5점은 예외적인 경우에만 부여됩니다.

3. 문제 커밋 식별 (revert, fix 패턴)

### 2. 평가 기준 엄격 적용

**별점 부여 규칙**:
- 3점 기본값으로 시작
- **증거 기반 가점**: 명확한 우수성 입증 시 +0.5~1.0
- **증거 기반 감점**: 명확한 문제 발견 시 -0.5~1.0
- 4점 이상: 업계 평균 대비 명확히 우수해야 함
- 5점: 거의 부여하지 않음 (탁월한 증거 필요)

### 3. 약점 의무화

**최소 요구사항**:
- 약점 5개 이상 필수
- 각 약점마다 구체적 개선 방법 제시
- 정량 지표로 뒷받침

### 4. 업계 표준 비교
