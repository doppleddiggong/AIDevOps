## 1. 프로젝트 개요

### 1.1 컨셉

**"PM 없이도 PM처럼 동작하는 가상 PM 매니저"**

기존 PM이 수동으로 작성하던 모든 관리 문서를 AI 기반으로 자동 생성하고, 프로젝트 진행 상황을 실시간으로 시각화하여 보고하는 솔루션입니다.

### 1.2 핵심 문제 인식

- **PM 부재**: 스타트업/신생 회사는 PM을 고용할 여력이 없음
- **문서 작업 부담**: 기존 PM도 반복적인 문서 작업에 시간 소비
- **정보 파편화**: Git, Slack, Jira, Discord 등 도구가 분산되어 통합 관리 어려움
- **진행도 추적 어려움**: 실시간 프로젝트 상황 파악이 어려움

### 1.3 솔루션

**AI 기반 자동화 + Git 로그 분석 + 통합 리포팅**

- Git 커밋 히스토리를 분석하여 자동으로 진행도 산출
- 간트차트, 마일스톤, 리스크 관리 문서 자동 생성
- 시각적 대시보드로 한눈에 프로젝트 상태 파악
- 클릭 몇 번으로 PM 리포트 완성

## 2. 핵심 가치 제안

### 2.1 사용자 가치

| 대상 | 제공 가치 |
|------|-----------|
| **PM 부재 팀** | PM 없이도 전문적인 프로젝트 관리 가능 |
| **주니어 PM** | PM 업무 가이드 및 학습 도구 |
| **시니어 PM** | 반복 작업 자동화로 전략적 업무에 집중 |
| **경영진/PD** | 실시간 프로젝트 현황 및 리스크 파악 |

### 2.2 차별화 포인트

1. **완전 자동화**: Git 로그만으로 모든 PM 문서 생성
2. **제로 러닝 커브**: 4지선다 폼 방식으로 MD 몰라도 사용 가능
3. **통합 생태계**: Discord, Email, 간트차트, CI/CD 연동
4. **SaaS 모델**: 설치형이 아닌 클라우드 기반 서비스
5. **레퍼런스 제공**: 실제 성공 사례 기반 템플릿 제공

## 3. 타겟 사용자

### 3.1 Primary Target

**PM이 없는 스타트업 개발팀 (5-20명)**

- 특징: 빠른 개발 속도, 문서 관리 부족, 커뮤니케이션 이슈
- 니즈: 최소 비용으로 프로젝트 관리 체계 구축
- 예산: 월 $50-200

### 3.2 Secondary Target

**주니어 PM (경력 1-3년)**

- 특징: PM 역할 수행 중이나 경험 부족
- 니즈: PM 업무 가이드, 문서 템플릿, 모범 사례
- 예산: 개인 구독 월 $20-50

### 3.3 Tertiary Target

**기존 PM (경력 3년+)**

- 특징: 다수 프로젝트 동시 관리, 시간 부족
- 니즈: 반복 작업 자동화, 대시보드 통합
- 예산: 팀 구독 월 $100-500

## 4. 핵심 기능

### 4.1 Phase 1: MVP (3개월)

#### 4.1.1 자동 리포트 생성

**입력**: Git 커밋 히스토리
**출력**: Markdown 기반 PM 리포트

- **진행도 리포트**
  - 기간별 커밋 분석
  - 개발자별 기여도
  - 모듈별 진행 상황

- **마일스톤 리포트**
  - 목표 대비 달성률
  - 지연 태스크 분석
  - 리스크 식별

- **주간/월간 리포트**
  - 주요 성과 요약
  - 이슈 및 해결 사항
  - 다음 주/월 계획

#### 4.1.2 간단한 폼 인터페이스

**"클릭 4번으로 리포트 완성"**

```
Step 1: 프로젝트 선택 (Git 저장소)
Step 2: 리포트 타입 선택 (주간/월간/마일스톤)
Step 3: 기간 선택 (날짜 범위)
Step 4: 생성 버튼 클릭 → 완료!
```

#### 4.1.3 기본 시각화

- 커밋 타임라인 차트
- 개발자별 기여도 파이 차트
- 모듈별 진행률 바 차트

---

### 4.2 Phase 2: 확장 기능 (6개월)

#### 4.2.1 간트차트 연동

**자동 일정 생성**

- Git 커밋 기반 실제 진행도 반영
- 예상 완료일 AI 예측
- 의존성 관계 시각화

**간트차트 툴 연동**

- Notion 간트차트 export
- Jira 연동
- Excel/Google Sheets export

#### 4.2.2 TODO 관리 시스템

**자동 태스크 추적**

- Git 이슈, 커밋 메시지에서 TODO 추출
- 우선순위 자동 배정 (AI 기반)
- 상태 관리: TODO → IN_PROGRESS → COMPLETE

**작업 LOSS 처리**

- 중단/취소된 작업 자동 감지
- LOSS 사유 기록 템플릿
- 재사용 가능한 작업 아카이빙

#### 4.2.3 시각적 대시보드

**캔바/피그마 스타일 인터랙티브 대시보드**

- 드래그 앤 드롭 위젯 배치
- 실시간 데이터 업데이트
- 발표자료용 export (PDF/PNG)

**주요 위젯**

- 프로젝트 건강도 게이지
- 마일스톤 진행 타임라인
- 리스크 알림 패널
- 팀원 활동 히트맵

## 5. 기술 스택

### 5.1 백엔드

- **언어**: Python 3.11+
- **프레임워크**: FastAPI
- **AI**: Claude API (Anthropic)
- **데이터베이스**: PostgreSQL
- **캐시**: Redis
- **작업 큐**: Celery

### 5.2 프론트엔드

- **프레임워크**: React + TypeScript
- **UI 라이브러리**: Tailwind CSS + shadcn/ui
- **차트**: Recharts, D3.js
- **간트차트**: react-gantt-timeline
- **상태 관리**: Zustand

### 5.3 인프라

- **클라우드**: AWS / GCP
- **컨테이너**: Docker + Kubernetes
- **CI/CD**: GitHub Actions
- **모니터링**: Grafana + Prometheus
- **로그**: ELK Stack

### 5.4 통합

- **Git**: GitHub API, GitLab API, Bitbucket API
- **Jenkins**: REST API
- **Firebase**: Admin SDK
- **Discord**: Webhook + Bot API
- **Email**: SendGrid API
